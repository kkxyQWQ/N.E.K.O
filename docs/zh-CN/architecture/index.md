# 架构概览

Project N.E.K.O. 采用**多进程微服务架构**构建，四个独立服务器通过 WebSocket、HTTP 和 ZeroMQ 消息进行协作，由统一的启动器（launcher）编排管理。

![架构图](/framework.svg)

## 四服务器设计

| 服务器 | 端口 | 入口文件 | 职责 |
|--------|------|---------|------|
| **主服务器** | 48911 | `main_server.py` | Web UI、REST API、WebSocket 聊天、TTS |
| **记忆服务器** | 48912 | `memory_server.py` | 语义召回、时间索引历史、记忆压缩、设置提取 |
| **监控服务器** | 48913 | `monitor.py` | 健康检查、启动器协调的配置端点 |
| **智能体服务器** | 48915 | `agent_server.py` | 后台任务执行（MCP、Computer Use、Browser Use、用户插件） |

主服务器是面向用户的入口。它提供 Web UI 服务，处理所有 REST API 请求，并维护用于实时语音/文字聊天的 WebSocket 连接。记忆服务器、智能体服务器和监控服务器是内部服务，由主服务器分别通过 HTTP、ZeroMQ 和 WebSocket 与之通信。

## 通信模式

```
┌──────────────────────────────────────────┐
│              主服务器 (:48911)             │
│                                          │
│  FastAPI ─── REST 路由（11 个路由）        │
│  WebSocket ─── LLMSessionManager         │
│  ZeroMQ PUB ───┐                         │
│  ZeroMQ PULL ──┼── AgentEventBridge      │
│  HTTP Client ──┤                         │
└────────────────┼─────────────────────────┘
                 │
     ┌───────────┼───────────┐
     │           │           │
     ▼           ▼           ▼
   记忆        智能体      监控
   服务器      服务器      服务器
  (:48912)   (:48915)   (:48913)
```

- **主服务器 <-> 记忆服务器**：通过 HTTP 请求存储/查询记忆、设置提取和记忆审阅
- **主服务器 <-> 智能体服务器**：通过 ZeroMQ 发布/订阅进行任务委派和结果流式传输；HTTP 代理标志和能力查询
- **主服务器 <-> 监控服务器**：通过 HTTP 进行健康检查、配置状态和启动器协调
- **启动器 -> 所有服务器**：进程管理、端口回退、优雅关闭

## 关键架构模式

### 会话热切换

`LLMSessionManager` 在当前会话仍然活跃时，于后台预先准备新的 LLM 会话。约 40 秒的周期控制会话生命周期 —— 到达阈值后，记忆被归档，新会话在后台预热。当前轮次结束时，系统无缝切换到预热好的会话，实现零停机。音频在过渡期间被缓存，之后以限速方式发送。

### 按角色隔离

每个角色（通过 `lanlan_name` 标识）拥有独立的：
- `LLMSessionManager` 实例
- 同步连接器线程
- WebSocket 锁
- 消息队列
- 关闭事件

角色可以在运行时动态添加或移除，无需重启 —— 现有活跃会话被保留，同时配置热重载。

### 异步/同步边界

FastAPI 处理器是异步的。TTS 合成在专用线程中运行，通过队列通信。音频处理使用 `soxr` 状态化重采样器（24kHz -> 48kHz）。ZeroMQ 事件桥运行后台接收线程，使用 NOBLOCK 发送和 ACK/重试机制。

### 启动锁

启动器使用平台特定的锁来防止重复启动器实例：
- **Windows**：命名互斥体 `Global\NEKO_LAUNCHER_STARTUP_LOCK`
- **POSIX**：系统临时目录中的文件锁

### 健康检查签名

所有服务器在健康端点中返回 `{"app": "N.E.K.O", "service": "..."}`。启动器使用此签名来区分 N.E.K.O. 进程和可能占用相同端口的其他服务。
